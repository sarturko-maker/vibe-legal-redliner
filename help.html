Vibe Legal Redliner

Getting started guide

What This Extension Does

Vibe Legal Redliner reviews your contracts using AI and produces a Microsoft Word document with tracked changes — just like a lawyer's markup. Your documents are processed entirely in your browser. Only the contract text is sent to your chosen AI provider for analysis.

Getting Started — API Key Setup

An API key is a password that lets the extension communicate with the AI service. You need one before you can analyze contracts.

Option A: Google Gemini (recommended to start)

  1. Go to aistudio.google.com and sign in with your Google account.
  2. Click "Create API key" and copy the key that appears.
  3. In the extension, go to Settings, select Google Gemini as the provider, and paste your key into the API Key field.
  4. Click Test Connection to confirm it works.

Option B: OpenRouter

  1. Go to openrouter.ai and create an account.
  2. Generate an API key from your dashboard.
  3. In the extension, go to Settings, select OpenRouter as the provider, and paste your key.
  4. Click Test Connection to confirm it works.
Remember API key toggle: When this is on (the default), your key is saved between browser sessions. Turn it off if you prefer to re-enter your key each time you open the browser. Either way, the extension never stores your key on any server — it stays in your browser.

Choosing a Model

Different AI models vary in speed, quality, and cost. You can change models at any time in Settings.

Google Gemini

The extension lists available Gemini models after you connect. Start with the default selection and adjust if needed. See current options on the Google AI pricing page.

OpenRouter

OpenRouter is a gateway that gives you access to models from many providers. After connecting, the extension will show available models. See the full list and pricing on the OpenRouter models page.

Using Playbooks

A playbook is a set of instructions that tells the AI how to review your contract. For example: "review this NDA from the disclosing party's perspective" or "flag aggressive liability terms in this supply agreement."

The extension comes with example playbooks to get you started — you can view, edit, or delete these at any time.

Creating your own playbook

  1. Go to the Playbooks page and click + New Playbook.
  2. Give it a name and a short description.
  3. Write your review instructions in the rules field.
  4. Click Save Changes, then Use for Review to make it active.
Tip: The more specific your playbook, the better the results. Include which party you represent, your negotiating position, and what terms matter most to you. For example, instead of "review this contract," try "review this SaaS agreement from the customer's perspective, flag any limitation of liability below 12 months of fees, and ensure data deletion is required within 30 days of termination."

Processing a Contract

  1. Select or create a playbook that matches your review needs.
  2. Upload a .docx file (Microsoft Word document) by clicking the upload area or dragging the file in.
  3. Click Analyze & Redline. The extension extracts the text, sends it to the AI, and generates tracked changes.
  4. When processing is complete, click Download Redlined Document.
  5. Open the downloaded file in Microsoft Word. You'll see tracked changes that you can accept or reject, just like a normal legal markup.
Note: Processing time depends on document length and the AI model. Typical contracts take 30–60 seconds. The AI analysis step is usually the longest part.

Privacy & Security

  • Documents are processed in your browser using WebAssembly — your files are never uploaded to any server.
  • Contract text (not the file itself) is sent to your chosen AI provider for analysis.
  • API keys are stored locally in your browser and never transmitted to us.
  • There are no analytics, no tracking, and no accounts.

Troubleshooting

"API key invalid"

Double-check that you copied the full key with no extra spaces. Make sure it hasn't expired, and verify you selected the correct provider (Gemini vs. OpenRouter) in Settings.

"Processing seems stuck"

Large documents can take up to 2 minutes. The AI analysis step is usually the longest part — the progress bar will update as each phase completes.

"Downloaded file won't open"

Make sure you open the file with Microsoft Word. Google Docs and LibreOffice may not display tracked changes correctly.

"No changes in the output"

Check that your playbook instructions are specific enough. Try one of the example playbooks first to confirm the extension is working, then adjust your custom playbook.
